git push


将本地代码库的更新推送到远程代码库。请参考：一张图了解git

  • git push [<repository> [<refspec>]]

    • repository是远程代码仓库，refspec = <src>:<dst>，src为本地分支名，dst为远程分支名。

  • git push -u origin <branch>

    • 将本地branch分支推送到远程代码仓库，-u（= --set-upstream）的意思是指定origin为默认主机，后面就可以直接git push不带参数了。

    之后 git push ， git push origin ， git push origin <branch>  是一个意思。

  • git push origin :<branch>，git push origin --delete <branch>

    • 删除指定的远程分支branch。

  • git push支持refspec，详见refspec

git checkout


切换分支操作。

  • git checkout <branch>

    • 1、<branch> 为空，空操作

    2、<branch> ①非当前分支 ②本地已有该分支 则切换到<branch>分支

    3、<branch> ①非当前分支 ②本地没有该分支 ③已有同名远程分支，则本地创建<branch>分支并追踪对应远程分支，等价于git checkout -b <branchA> --track <remote>/<branchB>

    4、<branch> ①非当前分支 ②本地没有该分支 ③没有同名远程分支，报error

  • git checkout <commit>

    • 以分离(detch)的方式切换到<commit>，这种方式可以用来进行amend修改操作

    在gerrit评审中，就提供了checkout的Download方式，当提交了很多评审或者评审很混乱时，就可以通过这种方式amend修改中间的某一次评审

origin


origin是远程仓库在本地的(默认)别名。

origin/master是远程仓库的master分支。

  • 如何查看origin

    • 查看.git/config文件，其中的[remote "origin"]项即是：

      • [remote "origin"]
      url = https://github.com/yang-an-china/easygit.git
      fetch = +refs/heads/*:refs/remotes/origin/*
    • 命令：git remote -v

      • $ git remote -v
      origin https://github.com/yang-an-china/easygit.git (fetch)
      origin https://github.com/yang-an-china/easygit.git (push)
    • 命令：git config --get remote.origin.url

      • $ git config --get remote.origin.url
      https://github.com/yang-an-china/easygit.git

  • origin是唯一的吗？

    • 如前所言，origin仅仅是默认的别名，当然就可以修改。

    假设我要为远程仓库增加一个mine的别名，有如下途径：

    • 直接修改.git/config文件

      • [remote "mine"]
      url = https://github.com/yang-an-china/easygit.git
      fetch = +refs/heads/*:refs/remotes/origin/*
    • 命令：git remote add mine https://github.com/yang-an-china/easygit.git

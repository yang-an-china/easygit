git reset


将当前HEAD复位到指定状态。一般用于撤销之前(例如git add，git commit等)的操作。

  • git reset [mode] <commit>

    • 将HEAD复位到commit处。（mode = --mixed(默认)，--soft，--hard，--merge，--keep）

    reset which? mixed soft hard merge keep
    index yes no yes yes yes
    working tree no no yes yes
    更新working tree中commit同HEAD的差异项
    保留working tree同index的差异项    		 yes
    更新working tree中commit同HEAD的差异项
    同时也更新working tree同index的差异项

  • git reset HEAD <file>

    • 当把错误的文件git add之后，可以通过此条命令回退；如果file为空，则回退所有文件。

  • git reset -- <file>

    • 重置某一个单独的文件。

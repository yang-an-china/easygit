refspec


refspec的全称是Reference Specification，全网都没有找到合适的翻译，个人觉得翻译成详细引用更加合适，反正大家记住这是git里面的一个专业术语就可以了。

它的作用是建立本地和远程分支之间的联系，用于fetch，pull和push等场合。可以参考：refs，一个讲解很详细的资料：The meaning of refs and refspecs

  • 格式：(+)<src>:<dst>

    • +号为可选，指：在非ff(fast-forward)模式下也强制更新引用。

    src 代表远程代码库的引用，一般填refs/heads/*。

    dst代表远程引用在本地的存储位置，一般填refs/remotes/origin/*。

  • .git/config文件

    • [remote "origin"]
    url = https://github.com/yang-an-china/easygit.git
    fetch = +refs/heads/*:refs/remotes/origin/*

  • git fetch [<repository> [<refspec>]]

    • 如下命令是一个意思。

    • git fetch origin develop
    • git fetch origin develop:develop
    • git fetch origin refs/heads/develop:refs/remotes/origin/develop

  • git push [<repository> [<refspec>]]

    • 如下命令是一个意思。

    • git push origin develop
    • git push origin develop:develop
    • git push origin refs/heads/develop:refs/remotes/origin/develop

    如下命令删除远程分支。

    • git push origin :develop
